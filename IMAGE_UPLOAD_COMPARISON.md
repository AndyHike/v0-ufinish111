# Відмінності між методами завантаження зображень

## До (Старий метод)

**Тільки посилання на фото**

```
[Admin Panel]
    ↓
[Input field "Featured Image URL"]
    ↓
[Зберігання посилання в БД]
    ↓
[Залежність від зовнішнього хосту]
```

Проблеми:
- Залежність від зовнішнього сервера
- Немає контролю над розміром
- Зображення може зникнути якщо хост видалить
- Немає оптимізації

## Після (Новий метод)

**Завантаження фото + посилання**

```
[Admin Panel]
    ↓
[ImageUploadField Component]
    ↓
[Вибір фото ЛІБ введення посилання]
    ↓
┌─────────────────┬──────────────────┐
│                 │                  │
v                 v                  v
[Завантажити]  [Посилання]      [Попередній перегляд]
        ↓
[Sharp обробка]
  - Конвертація в WebP
  - Стискання
  - Resize до 1200px
        ↓
[Cloudflare S3]
        ↓
[Публічна CDN URL]
        ↓
[Зберігання URL в БД]
```

## Технічні деталі

### Для завантажених фото:
- **Тип файлу**: WebP (оптимальний)
- **Розмір**: макс 1200px ширина
- **Якість**: 80%
- **Сховище**: Cloudflare R2 (S3-сумісне)
- **Доступ**: Публічна CDN URL

### Для посилань:
- Зберігаються як є
- Не обробляються
- Не завантажуються

## Користувацька точка зору

### Сценарій 1: Я хочу завантажити своє фото

1. Натискаю кнопку "Upload Image"
2. Вибираю фото з комп'ютера
3. Система автоматично:
   - Стискає
   - Конвертує в WebP
   - Завантажує на S3
   - Встановлює URL фото
4. Я бачу попередній перегляд

### Сценарій 2: Я хочу використати URL фото

1. Натискаю "Enter URL"
2. Копіюю посилання на фото
3. Система зберігає посилання як є

## Про збереження в БД

Нова колонка `featured_image_type` показує, звідки фото:
- `'uploaded'` - завантажено на S3
- `'url'` - зовнішнє посилання

Це дозволяє системі розрізняти типи і обробляти їх відповідно.

## Переваги нового методу

1. **Надійність** - ваші фото завжди на S3
2. **Контроль** - ви контролюєте розмір і якість
3. **Швидкість** - WebP + CDN = швидше
4. **Гнучкість** - ви можете вибрати завантаження або посилання
5. **Аналітика** - можна відслідковувати використання фото

## Як це впливає на фронтенд?

Прочитано зображення просто отримуватиме URL (не важливо завантажено чи посилання):

```jsx
<img 
  src={article.featured_image} 
  alt={article.title}
/>
```

Все працює автоматично!
