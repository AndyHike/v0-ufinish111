# Хаб статей про ремонт - Документація

## Огляд

Ви тепер маєте повністю функціональну систему для управління статтями про ремонт з SEO оптимізацією та багатомовною підтримкою.

## Як розпочати

### 1. Створення статті через адмін-панель

1. Перейдіть на `/admin/articles`
2. Клікніть на кнопку "Create Article"
3. Заповніть обов'язкові поля:
   - **Title** - назва статті
   - **Content** - контент у HTML форматі
   - **Featured Image URL** - посилання на зображення (опціонально)
4. Система автоматично генерує:
   - **Slug** - URL адреса статті (з назви)
   - **Meta Description** - опис для пошукових систем (перші 155 символів)
   - **Reading Time** - час читання (розраховується від кількості слів)
5. Виберіть опції:
   - **Featured** - показувати статтю на топі хабу
   - **Published** - публікувати статтю
6. Клікніть "Save Article"

### 2. Додання перекладів

1. Після створення статті, клікніть на вкладку мови (uk, en)
2. Клікніть "Add {Language} Translation"
3. Заповніть:
   - **Title** - переведена назва
   - **Content** - переведений контент
   - Система автоматично генерує Meta Description
4. Клікніть "Save Translation"

Переклади можна додавати/редагувати/видаляти відповідно.

### 3. Додання посилань на послуги

1. При редагуванні статті, внизу форми буде розділ "Related Services"
2. Виберіть послуги, які пов'язані зі статтею
3. При перегляді статті користувачем буде показана карточка із пропозицією послуги з посиланням на замовлення

## Фронтенд

### Хаб статей: `/articles`

- Список всіх опублікованих статей
- Featured статті показуються на топі
- Функція пошуку за назвою
- Карточка статті містить:
  - Зображення
  - Назву
  - Preview (перші 150 символів)
  - Час читання
  - Кількість переглядів

### Сторінка статті: `/articles/[slug]`

- Повний контент статті
- Час читання та кількість переглядів
- **Related Services** - карточки послуг для замовлення
- **Related Articles** - схожі статті
- Автоматичний підрахунок переглядів

## SEO оптимізація

### Автоматична генерація

1. **Meta Title** - `{article.title} | DeviceHelp`
2. **Meta Description** - перші 155 символів контенту
3. **Open Graph теги** - для соціальних мереж
4. **Hreflang теги** - для багатомовної пошукової оптимізації

### Структуровані дані (JSON-LD)

Кожна сторінка статті автоматично містить:

- **Article Schema** - інформація про статтю
- **BreadcrumbList Schema** - навігаційна цепочка
- **Organization Schema** - інформація про компанію

### Sitemap

- Динамічний sitemap включає всі статті
- Автоматичне додавання hreflang тегів для кожної мови
- Посилання на хаб статей для кожної мови

## API

### Публічні endpoints

```
GET  /api/articles                    - список статей (з пагінацією)
GET  /api/articles/[id]               - деталі статті
POST /api/articles/[id]/views         - збільшити лічильник переглядів
GET  /api/articles/[id]/services      - послуги пов'язані зі статтею
```

### Адмін endpoints

```
POST   /api/articles                           - створити статтю
PUT    /api/articles/[id]                      - редагувати статтю
DELETE /api/articles/[id]                      - видалити статтю

POST   /api/articles/[id]/translations         - додати переклад
GET    /api/articles/[id]/translations         - отримати переклади
PUT    /api/articles/[id]/translations/[locale] - редагувати переклад
DELETE /api/articles/[id]/translations/[locale] - видалити переклад

POST   /api/articles/[id]/services            - додати посилання на послугу
GET    /api/articles/[id]/services            - отримати послуги
DELETE /api/articles/[id]/services            - видалити посилання на послугу
```

## Бази даних

### Таблиця: `articles`

- `id` - UUID
- `slug` - унікальний URL адреса
- `title` - назва статті
- `content` - HTML контент
- `meta_description` - опис для пошуку
- `featured` - чи показувати на топі
- `published` - чи опублікована
- `view_count` - кількість переглядів
- `reading_time_minutes` - час читання
- `featured_image` - URL зображення
- `created_at` - дата створення
- `updated_at` - дата оновлення

### Таблиця: `article_translations`

- `id` - UUID
- `article_id` - FK до articles
- `locale` - мова (cs, uk, en)
- `title` - переведена назва
- `content` - переведений контент
- `meta_description` - переведений опис

### Таблиця: `article_service_links`

- `id` - UUID
- `article_id` - FK до articles
- `service_id` - FK до services
- `position` - порядок

## Особливості

- Автоматична генерація slug з назви
- Автоматичне розрахунок часу читання
- Автоматична генерація мета-опису
- Багатомовна підтримка (cs, uk, en)
- SEO оптимізація з JSON-LD схемами
- Динамічний sitemap з hreflang
- Лічильник переглядів для кожної статті
- Внутрішні посилання на послуги для конверсії

## Порядок роботи

1. Напишіть статтю в адмін-панелі
2. Додайте переклади для інших мов
3. Додайте посилання на відповідні послуги
4. Публікуйте статтю
5. Статя автоматично:
   - Додається до sitemap
   - Отримує JSON-LD схему
   - Індексується Google
   - Показується у пошуку і хабі

Користувачі читають статтю, бачать пов'язані послуги і можуть відразу замовити послугу або записатися на діагностику!
