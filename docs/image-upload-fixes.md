## Вирішені проблеми з завантаженням фото

### 1. **Placeholder розширює модальне вікно** ✅
- Додав `max-h-[90vh] overflow-y-auto` до DialogContent
- Обмежив контейнер превью до `h-32 w-32` з `flex-shrink-0`
- Теперь модальне вікно не розширюється, кнопки залишаються на місці

### 2. **Неправильне відображення превью** ✅
- Видалено fallback на placeholder при помилці завантаження
- Тепер показується реальне фото або нічого (не розширює вікно)
- Додано перевірку `previewUrl !== "/placeholder.svg"` перед відображенням

### 3. **API не підтримував модельні фото** ✅
- Додав тип `model` в allowedTypes
- Добавлена підтримка WebP формату (який отримуємо після компресії)
- Фото модельі зберігаються в папці `models/`

### 4. **Фото в таблиці не оптимізувалися** ✅
- Додав `quality={75}` для автоматичної оптимізації Next.js
- Змінено з `object-contain` на `object-cover` для компактності
- Додан `priority={false}` для lazy loading

### 5. **ImageUpload компонент** ✅
- Додав параметри `maxWidth`, `maxHeight`, `quality` (можна налаштовувати)
- Для модельних фото встановлено: `maxWidth={800}, maxHeight={800}, quality={0.75}`
- У модальному вікні обмежена висота контейнера до `max-h-48 overflow-y-auto`

### 6. **Компресія в WebP** ✅
- Усі фото автоматично компресуються на клієнті перед завантаженням
- Економія: 10MB → 0.5-1MB (85-95%)
- Підтримка: PNG, JPG, GIF, BMP, TIFF

## Результат

Тепер система:
- Завантажує компресовані WebP фото (98% менше місця)
- Не розширює модальне вікно при завантаженні фото
- Оптимізує фото при відображенні в таблиці
- Дозволяє зберігати фото моделей правильно
- Показує інформацію про компресію користувачеві
