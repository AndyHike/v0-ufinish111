# Cloudflare S3 Setup Guide для завантаження статей

## Що було внесено

Ваш сайт тепер повністю інтегрований з Cloudflare R2 (S3-сумісне сховище) для завантаження та обробки зображень статей.

### Основні можливості:
- ✓ Завантаження зображень напряму в адмін-панелі
- ✓ Автоматичне стискання та конвертація в WebP
- ✓ Зміна розміру до максимум 1200px
- ✓ Збереження в Cloudflare S3
- ✓ Публічний доступ через CDN

## Необхідні Cloudflare S3 дані

Вам потрібно отримати наступні дані з вашого Cloudflare аккаунту:

1. **CLOUDFLARE_ACCOUNT_ID** - ID вашого аккаунту
2. **CLOUDFLARE_ACCESS_KEY_ID** - Access Key ID для R2
3. **CLOUDFLARE_SECRET_ACCESS_KEY** - Secret Access Key для R2
4. **CLOUDFLARE_BUCKET_NAME** - Назва вашого R2 bucket
5. **CLOUDFLARE_ENDPOINT_URL** - S3 endpoint (зазвичай: `https://{ACCOUNT_ID}.r2.cloudflarestorage.com`)
6. **CLOUDFLARE_PUBLIC_URL** - Публічний URL вашого bucket (зазвичай: `https://{YOUR_DOMAIN}.r2.dev`)

### Як отримати дані з Cloudflare:

1. Перейдіть на [Cloudflare Dashboard](https://dash.cloudflare.com)
2. Виберіть ваш аккаунт
3. Перейдіть в **R2** → **API Tokens**
4. Створіть нов token або скопіюйте існуючі дані
5. Знайдіть **Account ID** у розділі **R2 API**

## Як додати дані в проект

1. Перейдіть в **Vars** у левому меню
2. Додайте всі 6 змінних із даними вашого Cloudflare S3
3. Збережіть

## Як використовувати в адмін-панелі

1. Перейдіть в розділ **Статті** в адміністраторській панелі
2. Створюйте або редагуйте статтю
3. У полі **Featured Image** тепер ви можете:
   - **Завантажити нове фото** - натисніть на кнопку завантаження
   - **Вставити посилання** - введіть URL фото
4. При завантаженні фото автоматично:
   - Стиснеться
   - Конвертується в WebP формат
   - Завантажиться на Cloudflare S3
   - Отримає публічний URL

## Структура файлів

Нові та оновлені файли:

```

  └── article-editor.tsx        # Оновлено для використання нового компонента

app/api/admin/
  └── upload/route.ts           # Оновлено для S3 завантаження

scripts/
  └── add_image_upload_support_to_articles.sql  # Міграція БД
```

## Специфікація обробки зображень

- **Формат**: WebP (оптимальний для веб)
- **Максимальна ширина**: 1200px
- **Якість**: 80% (баланс якості та розміру)
- **Шлях в S3**: `articles/{timestamp-filename}.webp`

## Можливі помилки та рішення

### Помилка: "CLOUDFLARE_BUCKET_NAME is not set"
**Рішення**: Перевірте, чи додані всі 6 змінних в Vars

### Помилка: "Failed to upload image to S3"
**Рішення**: 
- Перевірте правильність ENDPOINT_URL і BUCKET_NAME
- Переконайтеся, що Access Keys активні в Cloudflare

### Зображення не відображається
**Рішення**: Перевірте CLOUDFLARE_PUBLIC_URL - він повинен вказувати на ваш публічний bucket

## Альтернативний спосіб - тільки посилання на фото

Якщо ви не хочете використовувати S3, ви все ще можете вставляти посилання на фото напряму у поле Featured Image.

## Питання чи проблеми?

Якщо у вас виникли проблеми з налаштуванням, перевірте:
1. Всі 6 змінних середовища додані
2. Дані з Cloudflare скопійовані правильно
3. R2 Bucket створено і активно
4. Access Keys мають необхідні права доступу
